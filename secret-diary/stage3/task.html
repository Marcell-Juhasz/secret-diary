Description

Everyone makes mistakes. This situation could also happen with the user. Imagine that they mistype a word in the input text and save it. After that, they realize the typo in the diary and do not want to leave it as is. The user should be able to remove the last note from the diary, and this is what you need to implement in this stage using an "Undo" button.

This is not enough. What if the user accidentally taps the undo button after writing a really long text? To avoid such misclicks, an AlertDialog should appear that ask the user if they are sure in deleting the last note.

Objectives

1. Create a new Button with id attribute "btnUndo" and text set to "Undo".
2. On undo button click, an AlertDialog should be displayed with the following properties:

  • Title: "Remove last note"
  • Message: "Do you really want to remove the last writing? This operation cannot be undone!"
  • Positive Button: "Yes"
  • Negative Button: "No"

3. If the user selects Yes, then perform the action, remove the last writing from the diary. Otherwise, nothing should happen.
