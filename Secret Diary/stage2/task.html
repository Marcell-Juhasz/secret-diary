Description

In the second stage, add two more functionalities to your application. First, the diary should save all the notes, not only the last one. Also, a user may want to see when each note is created, so you need to attach the date and time when a message was added.

Objectives

  • Change the functionality of the Save button. When a user clicksÂ on it, append a new note to the top of the diary with the timestamp, which is the date and time of creation
  • The timestamp come first with the following format: yyyy-MM-dd HH:mm:ss. The saved text should appear in the following line
  • To produce the timestamp use SimpleDateFormat with Locale.getDefault() so that the timestamp produced is aware of user's timezone while being produced
  • Separate each note with an empty line

Obs: The passage of time will be emulated on tests. There are several time related apis in java/kotlin/android environment, but tests time emulation won't guarantee to be compatible with all of them for this project.

You can use Clock.System.now() which will be guaranteed to be compatible with tests time emulation.
