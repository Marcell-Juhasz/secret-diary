Description

Everyone makes mistakes. Users can mistype a word in the input text and save it. After that, they can realize they made a typo in the diary and fix it. In this stage, you need to implement the Undo button.

It is not enough. What if users accidentally click the undo button after composing a long entry? To avoid such misclicks, an AlertDialog should appear that ask the user if they are sure about deleting the last note.

Objectives

  1. Create a new Button with the id attribute of btnUndo and text set to Undo
  2. When users click on the Undo button, display an AlertDialog with the following properties:
    • Title: Remove last note
    • Message: Do you really want to remove the last writing? This operation cannot be undone!
    • Positive Button: Yes
    • Negative Button: No
    • When importing AlertDialog make sure to import android.app.AlertDialog
  3. If a user selects Yes, remove the last note from the diary. Otherwise, do nothing
  4. If a user selects Yes on the AlertDialog but the diary is empty, do nothing
